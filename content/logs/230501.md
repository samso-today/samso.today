# 목적 Objective

- 문서의 목적, 어떤 내용을 다룰 것인지 명확하게

# 배경 정보 Background

- 2~3문단에서 반 장 정도로 왜 이 신규 기능이 필요한지에 대해 설명
- 일련의 진행 상황, 풀고자 하는 문제, 앞으로의 방향성에 대해 이해할 수 있어야 한다

# 고객을 위해 어떤 일을 하는가? What job are you doing for the Customer?

- 목록 형태로 작성, 3~5가지의 항목이 나열
- 각 고객이 왜 해당 기능을 ‘고용’할지에 대해 짧고 명확하게 명시
- 1번부터 중요도에 따라 나열 “정확하게 어떤 음식을 주문할지 알고 들어온 고객은 실시간 배송 정보를 눈으로 확인하기 위해 새로운 지도 기능을 고용한다”

# 원칙 Guiding Principles

- 고객에게 선보이는 과정에서 결정을 내릴 때 잣대로 삼을 원칙을 나열
- 1번부터 중요도에 따라 명시, 주로 6개 내로 한정
- “고객은 음식 배달 현황을 파악하는 걸 가장 중요하게 생각한다. 배달 현황을 파악하는 데 도움이 안 되는 지도 기능은 철저하게 배제한다.”

# 목표 Goals

- 어떤 목표를 달성할지 설명
- 목표는 무조건 수치화되어 있어야 한다
- 가설을 증명하기 위해 사용되는 수치가 그대로 기입될 수 있다
- 2~3개로 한정

# 주요 지표 Key Metrics

- 목표에 사용되는 지표를 포함하여, 기능이 고객을 위해 제대로 된 목적을 수행하고 있는지 나타낼 수 있는 지표를 3~4가지 선정
